# 🪟 Windows Architecture for SOC Analysts

> **"You don't need to know every Windows component—you need to know the components attackers abuse and defenders investigate."**

---

# 🎯 Chapter Objectives

After completing this chapter, you will be able to:

- Understand the core components of Windows.
- Recognize important Windows processes.
- Understand how users interact with the operating system.
- Identify common attacker targets.
- Investigate Windows-based alerts with greater confidence.

---

# 📖 Why Should a SOC Analyst Learn Windows Architecture?

Every alert generated by an EDR, SIEM, antivirus, or Sysmon originates from a Windows component.

For example:

- A malicious process starts.
- PowerShell executes a suspicious command.
- A new Windows service is created.
- A scheduled task is added.
- A user logs in remotely.
- A DLL is loaded into memory.

To investigate these events effectively, you need to understand how Windows is structured.

This chapter focuses only on the Windows architecture that is relevant to SOC analysts.

---

# 🏗️ High-Level Windows Architecture

A simplified view of Windows looks like this:

```
+--------------------------------------+
|             User                     |
+--------------------------------------+
                │
                ▼
+--------------------------------------+
|        Applications & Services       |
+--------------------------------------+
                │
                ▼
+--------------------------------------+
|          Windows Operating System    |
+--------------------------------------+
                │
                ▼
+--------------------------------------+
|     Hardware (CPU, RAM, Disk, NIC)   |
+--------------------------------------+
```

Every action performed by a user or an application eventually interacts with the Windows operating system and underlying hardware.

SOC analysts primarily investigate activity occurring in the **Applications & Services** layer and the **Operating System** layer.

---

# 🔗 How Everything Connects

A typical Windows activity follows this flow:

```
User

↓

Application

↓

Windows Process

↓

Operating System

↓

Event Log / Sysmon

↓

SIEM / EDR Alert

↓

SOC Investigation
```

This is the path most security events follow before appearing on a SOC analyst's dashboard.

---

# 💡 SOC Insight

When investigating an alert, ask yourself:

- Which process generated the activity?
- Which user executed it?
- Was it expected?
- Which Windows component was involved?
- Which logs contain evidence?

Understanding these relationships helps you investigate incidents faster and with greater confidence.
