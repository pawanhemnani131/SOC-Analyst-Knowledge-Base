# 🎯 Why SOC Analysts Should Learn the OSI Model

A SOC Analyst doesn't just investigate alerts—they investigate **network communication**.

Every alert generated by a firewall, IDS/IPS, SIEM, or EDR is related to one or more layers of the OSI Model.

Understanding the OSI Model helps you answer questions like:

- 🔍 Where did the attack happen?
- 🌐 Which protocol was used?
- 📂 Which logs should I investigate?
- 🚨 Which security device generated the alert?
- 🛡️ How can this attack be detected or prevented?

Without understanding the OSI Model, analyzing network traffic becomes much more difficult.

---

# 🌐 What is the OSI Model?

The **Open Systems Interconnection (OSI) Model** is a conceptual framework that describes how data travels from one device to another over a network.

Instead of treating communication as one large process, the OSI Model divides it into **seven layers**, where each layer has a specific responsibility.

Think of the OSI Model as a **step-by-step delivery process**.

For example, when you send a message from your laptop to a website:

1. Your browser creates the request.
2. The request is prepared for transmission.
3. A communication session is established.
4. Data is divided into smaller segments.
5. IP addresses are added.
6. Frames are created using MAC addresses.
7. The data is converted into electrical, optical, or wireless signals.

The receiving device performs these steps in reverse until the original message is reconstructed.

This layered approach makes networking standardized, easier to troubleshoot, and more secure.

---

# 📖 History

Before the OSI Model existed, different computer manufacturers developed their own proprietary networking standards.

This created major compatibility issues because systems from different vendors often couldn't communicate with each other.

To solve this problem, the **International Organization for Standardization (ISO)** introduced the **OSI Model** in **1984** as a universal reference model for network communication.

Although modern networks primarily use the **TCP/IP Model**, the OSI Model remains the global standard for learning, designing, troubleshooting, and securing networks.

For cybersecurity professionals, it provides a common language to describe attacks and defenses.

---

# 🎯 Why Was It Created?

The OSI Model was designed to make network communication:

### 🌍 Standardized
Devices from different manufacturers can communicate using common networking principles.

### 🔄 Modular
Each layer performs a specific task without depending on how the other layers are implemented.

### 🔧 Easier to Troubleshoot
Network engineers and SOC analysts can isolate problems to a specific layer instead of troubleshooting the entire communication process.

### 🛡️ Better for Security
Security controls such as firewalls, IDS/IPS, VPNs, and EDR solutions operate at different OSI layers. Understanding the model helps defenders know where to monitor and where attacks occur.

### 📚 Easier to Learn
Breaking communication into seven layers simplifies complex networking concepts and provides a structured approach to learning.

Imagine ordering a product from an online shopping website.

- 🛒 You place the order (Application Layer).
- 📦 The item is packed (Presentation Layer).
- 📞 The courier confirms your delivery (Session Layer).
- 🚚 The package is divided into shipments if necessary (Transport Layer).

- 🏠 The local delivery center identifies your exact house (Data Link Layer).
- 🚪 The courier physically delivers the package to your doorstep (Physical Layer).

Just as every step is necessary for successful delivery, every OSI layer plays an important role in successful network communication.

# 🏗️ The Seven Layers of the OSI Model

Communication between two devices doesn't happen all at once.

Instead, data passes through **seven different layers**, each responsible for a specific task.

When data leaves your computer, it travels from **Layer 7 down to Layer 1**.

When the destination receives it, the process happens in reverse—from **Layer 1 back to Layer 7**.

         Sender
           │
           ▼
7️⃣ Application
6️⃣ Presentation
5️⃣ Session
4️⃣ Transport
3️⃣ Network
2️⃣ Data Link
1️⃣ Physical
────────────────────────
Transmission Medium
────────────────────────
1️⃣ Physical
2️⃣ Data Link
3️⃣ Network
4️⃣ Transport
5️⃣ Session
6️⃣ Presentation
7️⃣ Application
           ▲
           │
       Receiver

Each layer adds its own information before passing the data to the next layer. This process is called **Encapsulation**.

At the receiving side, every layer removes its corresponding information until the original data is reconstructed. This process is called **Decapsulation**.

# 7️⃣ Layer 7 — Application Layer

## 🎯 Primary Function

The Application Layer provides network services directly to the end user and allows applications to communicate over the network.

It is the closest layer to the user and is where web browsers, email clients, messaging applications, cloud services, and many other applications interact with the network.

This layer **does not refer to the application itself**, but rather the networking services that the application uses.

---

That request starts at the **Application Layer**.

Every time you:

# 6️⃣ Layer 6 — Presentation Layer

## 🎯 Purpose

The **Presentation Layer** is responsible for preparing data so that it can be correctly understood by the receiving device.

Think of it as the **translator** of the OSI Model.

It handles:

- Data Formatting
- Data Encryption
- Data Decryption
- Data Compression
- Data Decompression
- Character Encoding

Without this layer, two systems using different data formats would not be able to understand each other.

---

## 💡 Simple Analogy

Imagine two people speaking different languages.

One speaks English.

The other speaks Japanese.

Neither understands the other.

A translator sits between them and converts each sentence into the correct language.

The Presentation Layer works exactly the same way.

It ensures that data sent by one device can be correctly interpreted by another device.

---

## 🌐 Technical Explanation

When an application sends data, the Presentation Layer prepares it before transmission.

Depending on the application, it may:

- Encrypt sensitive information
- Compress large files
- Convert text into a standard encoding format
- Convert multimedia into supported formats

The receiving Presentation Layer performs the reverse process by decrypting, decompressing, and decoding the received data.

---

## 📦 Protocol Data Unit (PDU)

```
Data

```

No IP addresses or port numbers have been added yet.

1. Encrypts your HTTP request using TLS.
2. Validates the server's digital certificate.
3. Establishes a secure encrypted session.
4. Protects sensitive information from being read during transmission.

Without this layer, usernames, passwords, cookies, and other sensitive information would travel across the network in plain text.

- Capturing sensitive information before encryption or after decryption

Because encrypted traffic is difficult to inspect, attackers often hide malicious communication inside legitimate HTTPS sessions.

**Answer:** Yes. Analysts can examine metadata such as destination domains, certificate details, JA3 fingerprints, connection timing, process execution, DNS lookups, and traffic patterns even when the payload is encrypted.

# 5️⃣ Layer 5 — Session Layer

## 🎯 Purpose

The **Session Layer** is responsible for establishing, managing, maintaining, and terminating communication sessions between two devices.

A **session** is an active conversation between two systems. This layer ensures that both parties can exchange data in an organized and synchronized manner.

It also handles session recovery, synchronization, and graceful termination when communication ends.

---

## 💡 Simple Analogy

Imagine you're on a phone call with a friend.

📞 You dial their number.

📞 They answer.

🗣️ You both talk.

☎️ When the conversation is over, one of you hangs up.

That entire conversation is called a **session**.

The Session Layer performs the same function for network communication.

It creates the conversation, keeps it active, and closes it when it's finished.

---

## 🌐 Technical Explanation

Many modern applications require users to remain authenticated while interacting with services.

For example:

- Gmail
- Facebook
- GitHub
- Microsoft 365
- Banking Applications
- Online Shopping

When you log in, the application creates a **session**.

Instead of asking for your username and password every time you click a button, the application stores a **Session ID** or **Authentication Token**.

As long as the session remains valid, the server recognizes you as an authenticated user.

---

## 🔄 Normal Communication

A user signs in to Microsoft 365.

The server validates the credentials and issues a secure session token.

Each subsequent request includes that token instead of the user's password.

When the user signs out or the session expires, the server invalidates the token and the session ends.

This process improves both usability and security.

If an attacker obtains a session cookie or authentication token, they may gain access without needing the user's credentials.

This is why session-based attacks are common after phishing or browser compromise.

# 4️⃣ Layer 4 — Transport Layer

## 🎯 Purpose

The **Transport Layer** is responsible for delivering data reliably between applications running on different devices.

It ensures that data reaches the correct application using **port numbers** while managing reliability, sequencing, flow control, and error recovery.

Unlike the Network Layer, which delivers packets between devices, the Transport Layer ensures the right application on the destination device receives the data.

---

## 💡 Simple Analogy

Imagine sending a package to a large office building.

The **building address** gets the package to the correct building.

However, there are hundreds of employees inside.

The **office number** tells the courier exactly who should receive the package.

Think of:

- IP Address = Building Address
- Port Number = Office Number

Without port numbers, data would reach the correct computer but not the correct application.

---

## 🌐 Technical Explanation

The Transport Layer provides two communication methods:

### TCP (Transmission Control Protocol)

TCP is a **connection-oriented protocol**.

Before any data is transmitted, both devices establish a connection using the famous **Three-Way Handshake**.

TCP guarantees:

- Reliable delivery
- Correct packet ordering
- Error detection
- Retransmission of lost packets
- Flow control
- Congestion control

TCP is used whenever reliability is more important than speed.

Examples:

- HTTPS
- SSH
- FTP
- SMTP
- SMB
- RDP

---

### UDP (User Datagram Protocol)

UDP is a **connectionless protocol**.

It sends data without establishing a connection.

There is:

- No handshake
- No acknowledgements
- No retransmission
- No ordering

UDP is much faster than TCP but less reliable.

Examples:

- DNS
- VoIP
- Video Streaming
- Online Gaming
- DHCP
- SNMP
- NTP

---

## 📦 Protocol Data Unit (PDU)

At Layer 4, the PDU depends on the protocol.

| Protocol | PDU |
|----------|-----|
| TCP | Segment |
| UDP | Datagram |

This is the first layer where **port numbers** are introduced.

---

# 🤝 TCP Three-Way Handshake

Every TCP connection begins with a handshake.

```text
Client                      Server

SYN ----------------------->

        <------------------- SYN-ACK

ACK ----------------------->
```

### Step 1

Client sends **SYN**

"I want to communicate."

---

### Step 2

Server replies **SYN-ACK**

"I received your request."

---

### Step 3

Client sends **ACK**

"Connection established."

Now data transfer begins.

---

# 🔚 TCP Four-Way Termination

When communication ends:

```text
FIN

ACK

FIN

ACK
```

The connection closes gracefully.

# 3️⃣ Layer 3 — Network Layer

## 🎯 Purpose

The **Network Layer** is responsible for delivering data between different networks.

Its primary job is to determine **where data should go** by using **IP addresses** and selecting the best route to reach the destination.

Unlike the Transport Layer, which delivers data to the correct application, the Network Layer ensures that packets reach the correct device.

Without the Network Layer, communication beyond the local network would not be possible.

---

## 💡 Simple Analogy

Imagine sending a package to a friend living in another city.

The courier first needs to know:

- Which country?
- Which state?
- Which city?
- Which street?

This address determines the route the package will follow.

The Network Layer works exactly the same way.

Instead of street addresses, it uses **IP addresses** to determine where data should travel.

---

## 🌐 Technical Explanation

The Network Layer adds **logical addressing** to data by attaching source and destination IP addresses.

It also performs:

- Routing
- Packet forwarding
- Fragmentation
- Path selection
- TTL (Time To Live) handling

Routers operate primarily at this layer and decide how packets travel across interconnected networks.

---

## 📦 Protocol Data Unit (PDU)

At Layer 3, the PDU is called a:

```
Packet
```

Each packet contains:

- Source IP Address
- Destination IP Address
- Protocol Information
- Time To Live (TTL)
- Fragmentation Details

- Cloud Routers

## 🔄 Normal Communication

When you browse a website:

1. DNS resolves the domain name.
2. Your computer creates an IP packet.
3. The packet receives a source IP and destination IP.
4. The router determines the best route.
5. The packet travels through multiple networks before reaching the destination.

Every router along the path examines the destination IP address and forwards the packet accordingly.

# 2️⃣ Layer 2 — Data Link Layer

## 🎯 Purpose

The **Data Link Layer** is responsible for **node-to-node communication** within the same local network (LAN).

Its primary responsibilities include:

- Physical (MAC) Addressing
- Framing
- Error Detection
- Media Access Control
- Switching

Unlike the Network Layer, which routes traffic between networks using IP addresses, the Data Link Layer delivers frames between devices on the **same network segment**.

---

## 💡 Simple Analogy

Imagine an apartment building.

The **building's address** (IP Address) gets you to the correct building.

Once inside, you still need the **apartment number** (MAC Address) to deliver the package to the correct resident.

Layer 3 gets you to the building.

Layer 2 gets you to the correct apartment.

---

## 🌐 Technical Explanation

Every network device has a unique **MAC Address (Media Access Control Address)**.

Example:

```
00:1A:2B:3C:4D:5E
```

When two devices communicate on the same LAN:

1. The sender first discovers the destination MAC Address using **ARP**.
2. The packet is encapsulated inside an Ethernet Frame.
3. The switch forwards the frame using its MAC Address Table (CAM Table).

Unlike routers, switches make forwarding decisions using MAC addresses.

---

## 📦 Protocol Data Unit (PDU)

The PDU at Layer 2 is called a:

```
Frame
```

An Ethernet Frame contains:

- Source MAC Address
- Destination MAC Address
- EtherType
- Payload
- Frame Check Sequence (FCS)

---

## 📡 Common Protocols

| Protocol | Purpose |
|----------|---------|
| Ethernet | LAN Communication |
| ARP | Resolve IP → MAC |
| STP | Prevent Switching Loops |
| VLAN (802.1Q) | Network Segmentation |
| LLDP | Device Discovery |
| PPP | Point-to-Point Communication |

---

## 💻 Devices Operating at Layer 2

- Network Switches
- Wireless Access Points
- Network Interface Cards (NICs)
- Bridges
- Hypervisors (Virtual Switches)

---

## 🔄 Normal Communication

A user wants to access a shared printer on the same office network.

Before sending data:

1. The computer checks its ARP Cache.
2. If the MAC Address is unknown, it sends an ARP Request.
3. The printer replies with its MAC Address.
4. The switch forwards Ethernet Frames to the correct device.

This process is completely transparent to the user.

---

# 1️⃣ Layer 1 — Physical Layer

## 🎯 Purpose

The **Physical Layer** is the foundation of the OSI Model. It is responsible for transmitting raw bits between devices through physical media.

Unlike the upper layers, the Physical Layer does not understand IP addresses, MAC addresses, ports, or protocols. It simply converts digital data into electrical, optical, or radio signals and transmits them across the communication medium.

Without the Physical Layer, no network communication would exist.

---

## 💡 Simple Analogy

Imagine two people talking over a walkie-talkie.

One person speaks into the microphone.

The voice is converted into radio waves.

The other walkie-talkie receives those radio waves and converts them back into sound.

The Physical Layer performs a similar function by converting digital information into signals that travel across cables, fiber optics, or wireless media.

---

## 🌐 Technical Explanation

The Physical Layer defines how data is physically transmitted between devices.

It includes:

- Electrical Signals
- Optical Signals
- Radio Frequency (RF) Signals
- Cables
- Connectors
- Network Interface Hardware
- Wireless Transmission

This layer focuses on **how bits travel**, not **what those bits contain**.

---

## 📦 Protocol Data Unit (PDU)

The PDU at Layer 1 is called:

```
Bits
```

Example:

```
01001010 01100001 01100011 01101011
```

These binary values are transmitted as electrical, optical, or wireless signals.

---

## 📡 Common Standards & Technologies

| Technology | Purpose |
|------------|---------|
| Ethernet (Physical Standards) | Wired Communication |
| Fiber Optics | High-Speed Communication |
| Wi-Fi (802.11) | Wireless Networking |
| Bluetooth | Short-Range Communication |
| NFC | Near Field Communication |
| USB | Device Connectivity |
| Serial Communication | Device Communication |

---

## 💻 Devices Operating at Layer 1

- Network Interface Cards (NICs)
- Hubs
- Repeaters
- Cables
- Fiber Optic Links
- Wireless Access Points (Radio Transmission)
- Connectors
- Patch Panels
- SFP Modules

## 🔄 Normal Communication

A user opens a website.

After all upper layers prepare the data, the Physical Layer converts binary information into signals.

Depending on the network medium, those signals travel as:

- Electrical signals over copper cables
- Light pulses through fiber optics
- Radio waves over Wi-Fi

The receiving device converts those signals back into binary data for the upper layers to process.
